# 좋은 승인 요청은 무엇이 다른가

## 먼저 떠올릴 장면

슬랙에 승인 요청이 하나 올라온다.
버튼은 두 개다. `승인`, `거절`.
근데 정작 왜 이 결론이 나왔는지, 승인하면 무슨 일이 일어나는지, 거절하면 어디로 돌아가는지는 안 보인다.

이런 화면을 보면 사람은 판단하는 게 아니라 추측하게 된다.
좋은 승인 UX는 사람을 오래 붙잡아 두는 화면이 아니라, 사람의 판단 시간을 줄여 주는 화면이어야 한다.

## 승인 요청에 꼭 있어야 할 것

- 지금 시스템이 무엇을 하려는지
- 왜 그렇게 판단했는지
- 다른 선택지는 무엇인지
- 승인하면 어떤 결과가 나는지
- 거절하면 어디로 돌아가는지

쉽게 말하면 이거다.

- 지금 무슨 일이 벌어지려는지
- 왜 그런 판단이 나왔는지
- 내가 눌렀을 때 무슨 일이 생기는지

## 나쁜 승인 요청의 모습

- "승인하시겠습니까?"만 있다
- 판단 근거가 없다
- 거절했을 때 시스템이 어떻게 되는지 모른다
- 이전 이력이나 맥락이 보이지 않는다

이런 요청은 사람을 검토자로 쓰는 게 아니라,
책임 전가용 버튼 누르는 사람으로 만든다.

## 실무 감각 한 줄

승인 UX는 사람을 느리게 만드는 화면이 아니라, 사람의 판단을 빠르게 만드는 화면이어야 한다.
