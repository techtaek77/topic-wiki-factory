# 어디에 사람 확인을 넣을까

## 좋은 체크포인트의 기준

사람 확인은 "중요해 보여서" 넣는 게 아니라, 아래 중 하나가 강할 때 넣는 편이 좋다.

- 틀렸을 때 비용이 크다
- 맥락 해석이 애매하다
- 정책 / 법 / 브랜드 리스크가 걸려 있다
- 되돌리기 어렵다
- 모델 확신도가 낮거나 근거가 빈약하다

## 나쁜 체크포인트의 패턴

- 모든 단계마다 승인 버튼을 넣는다
- 승인자가 판단할 정보 없이 결과만 보여 준다
- 자동화가 거의 끝난 뒤, 이미 피해가 난 다음에 승인 요청을 띄운다

## 작게 시작하는 방법

1. 사고 비용이 큰 1개 단계만 고른다.
2. 그 단계에서 필요한 근거 3개만 정한다.
3. 승인 / 거절 / 보류의 다음 동작을 미리 정의한다.
4. 실제 거절 사유를 기록해 다음 기준 개선에 쓴다.
