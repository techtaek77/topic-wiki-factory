# 어디에 사람 확인을 넣을까

## 먼저 떠올릴 순간

자동화가 거의 다 끝났다.
분류도 했고, 추천도 했고, 이제 발송이나 승인만 남았다.
그런데 그 시점에서야 사람이 "잠깐, 이건 아닌데?"를 발견한다.

이건 체크포인트가 없었던 게 아니라, 너무 늦게 있었던 경우에 가깝다.
좋은 체크포인트는 사람을 많이 부르는 구조가 아니라, 사고가 나기 직전에 정확히 멈춰 세우는 구조다.

## 좋은 체크포인트의 기준

사람 확인은 "중요해 보여서" 넣는 게 아니라, 아래 중 하나가 강할 때 넣는 편이 좋다.

- 틀렸을 때 비용이 크다
- 맥락 해석이 애매하다
- 정책 / 법 / 브랜드 리스크가 걸려 있다
- 되돌리기 어렵다
- 모델 확신도가 낮거나 근거가 빈약하다

쉽게 기억하면 이렇다.

- 틀리면 비싼가
- 애매해서 설명이 필요한가
- 지금 멈추지 않으면 되돌리기 힘든가

이 셋 중 하나라도 강하면 체크포인트 후보가 된다.

## 나쁜 체크포인트의 패턴

- 모든 단계마다 승인 버튼을 넣는다
- 승인자가 판단할 정보 없이 결과만 보여 준다
- 자동화가 거의 끝난 뒤, 이미 피해가 난 다음에 승인 요청을 띄운다

특히 마지막 줄이 중요하다.
사람 확인은 "사후 감탄사"가 아니라 "사전 브레이크"여야 한다.

## 작게 시작하는 방법

1. 사고 비용이 큰 1개 단계만 고른다.
2. 그 단계에서 필요한 근거 3개만 정한다.
3. 승인 / 거절 / 보류의 다음 동작을 미리 정의한다.
4. 실제 거절 사유를 기록해 다음 기준 개선에 쓴다.

## 기억에 남는 문장으로 정리하면

체크포인트는 승인 버튼을 심는 위치가 아니라,
사람 판단이 가장 값어치 있어지는 순간을 고르는 일이다.
