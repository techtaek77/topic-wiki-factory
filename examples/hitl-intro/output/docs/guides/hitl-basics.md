# HITL가 뭐고 왜 쓰는가

## 한 줄로

HITL는 자동화 흐름 중 중요한 판단 지점에 사람 검토를 넣어, 속도는 유지하면서 사고 비용을 줄이려는 설계 방식이다.

## 왜 필요한가

자동화는 빠르지만, 애매한 상황에서 자신감 있게 틀릴 수 있다.
반대로 사람은 느리지만 맥락과 예외를 읽는 데 강하다.
HITL는 둘 중 하나를 고르는 게 아니라, 각자의 강점을 쓰는 설계다.

## 어디서 특히 자주 쓰나

- 고객에게 외부 메시지를 보내는 단계
- 돈이 오가는 승인 단계
- 정책 위반 가능성이 있는 콘텐츠 처리
- 모델이 낮은 확신도를 보이는 분기
- 시스템이 여러 선택지 중 하나를 추천하지만 책임 있는 결정은 사람이 내려야 하는 단계

## 초보자가 자주 하는 오해

- 승인 단계를 많이 넣으면 더 안전하다
- 사람 검토자는 그냥 마지막 버튼만 누르면 된다
- AI 확신도 점수 하나만으로 사람 확인 필요 여부를 정할 수 있다

이 셋 다 반쯤만 맞다.
HITL는 버튼 개수보다 "어떤 정보와 기준으로 사람이 개입하느냐"가 더 중요하다.
