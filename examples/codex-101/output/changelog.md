# 변경 이력

> Codex 같은 코딩 에이전트는 버튼 이름보다 작업 습관이 더 오래 남는다. 이 문서는 "지금 어떤 흐름을 기준선으로 보면 되는지"를 짧게 확인하는 용도다.

## 2026-03-10 - 샘플 위키 기준선

- 첫 작업 흐름을 `폴더 열기 -> 상태 확인 -> 브랜치 생성 -> 작은 요청 -> diff 확인 -> commit/push` 순서로 정리했다.
- 입문자가 가장 먼저 읽을 문서로 `quick-start`, `agents-and-context`, `review-vs-implementation`을 묶었다.
- 변동성이 큰 제품 UI 설명보다 오래 가는 협업 습관과 요청 방식에 초점을 맞췄다.

## 다음에 언제 다시 보면 되나

- 툴 사용 흐름이 크게 바뀌었을 때
- 팀 규칙이 바뀌어 branch, review, commit 기준이 달라졌을 때
- `quick-start`나 `faq`가 현재 작업 습관과 어긋나기 시작했을 때
