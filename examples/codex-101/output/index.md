# Codex 101

> 코딩 에이전트에게 일을 맡기고, 결과를 검토하고, 다시 이어 가는 기본 루프를 배우는 입문 위키다.

## 왜 보나

코딩 에이전트는 빠르지만, 사람이 기대치를 안 잡아 주면 빠르게 엉뚱한 곳으로 갈 수도 있다.
그래서 입문 단계에서는 "무엇을 맡기면 좋은지", "어디서 review를 걸어야 하는지", "문맥을 어떻게 넘길지"를 먼저 익히는 편이 훨씬 덜 아프다.

## 먼저 보면 좋은 것

- [prerequisite-map](prerequisite-map.md) - 에이전트와 일하기 전에 어떤 감각이 있으면 좋은지 먼저 본다.
- [quick-start](docs/guides/quick-start.md) - 첫 작업을 가장 짧은 루프로 한 번 따라간다.

## 이 위키를 보는 순서

1. [quick-start](docs/guides/quick-start.md)로 작업 흐름을 먼저 잡는다.
2. [agents-and-context](docs/concepts/agents-and-context.md)로 역할 분리와 문맥 전달 감각을 익힌다.
3. [review-vs-implementation](docs/concepts/review-vs-implementation.md)로 언제 구현을 맡기고 언제 검토를 요청할지 구분한다.
4. [glossary](glossary.md)와 [faq](faq.md)로 자주 나오는 표현을 정리한다.
