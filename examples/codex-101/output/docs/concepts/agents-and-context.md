# Agents and Context

> 코딩 에이전트에게 일을 잘 맡기려면 역할과 문맥을 같이 설계해야 한다.

---

## 왜 필요한가

에이전트는 똑똑해 보여도, 지금 뭘 하려는지와 어디까지 건드려도 되는지 모르면 금방 산으로 간다.
그래서 "무슨 역할인지"와 "어떤 배경을 봐야 하는지"를 같이 넘기는 게 기본기다.

- 없으면: 구현, 리뷰, 설명이 한 요청에 섞여 결과가 흐려진다.
- 있으면: 작업 범위와 기대 결과가 분명해져 재작업이 줄어든다.
- 비유: 일을 잘하는 사람도 회의 제목 없이 회의실에 넣으면 당황하는 것과 비슷하다.

## 어떻게 적용하는가

작업을 맡길 때는 최소한 아래 네 가지를 같이 넘기면 좋다.

1. 목표: 무엇을 바꾸려는지
2. 범위: 어떤 파일이나 모듈을 건드리는지
3. 금지선: 손대면 안 되는 것
4. 검증: 끝나면 무엇을 확인할지

### 핵심 포인트

- 역할이 다르면 요청도 분리한다.
- 긴 배경보다 필요한 맥락만 압축해서 주는 편이 낫다.
- 상태 파일, 스펙, 최근 변경처럼 진짜 중요한 자료를 먼저 준다.

### 자주 하는 실수

- “대충 다 고쳐줘”처럼 범위를 안 줌 -> 파일/목표/제약을 같이 적는다.
- review와 implementation을 한 번에 요청함 -> 먼저 하나의 목적만 명확히 잡는다.
