# 시작 전에 알아야 할 것

> Codex 같은 코딩 에이전트는 마법사가 아니라 협업자에 가깝다. 그래서 기술 자체보다 "일을 어떻게 나누는지" 감각이 먼저 필요하다.

## 필수

| 개념 | 한 줄 설명 | 추천 자료 |
|------|-----------|---------|
| 작은 작업 단위 | 큰 부탁보다 검증 가능한 작은 부탁이 훨씬 잘 통한다. | [quick-start](docs/guides/quick-start.md) |
| Review 습관 | 결과를 바로 믿지 말고 위험한 부분부터 확인하는 태도다. | [review-vs-implementation](docs/concepts/review-vs-implementation.md) |

## 권장

| 개념 | 한 줄 설명 | 추천 자료 |
|------|-----------|---------|
| Context 전달 | 필요한 배경과 제한을 짧고 분명하게 넘기는 감각이다. | [agents-and-context](docs/concepts/agents-and-context.md) |
| Git 기본기 | 브랜치, 커밋, diff를 읽는 습관이 있으면 훨씬 편하다. | [faq](faq.md) |

## 몰라도 됨

이 위키를 보면서 같이 익혀도 되는 것들:

- 세부 모델 차이
- 병렬 오케스트레이션 설계
- 고급 CI 자동화
