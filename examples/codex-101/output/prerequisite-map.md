# 시작 전에 알아야 할 것

> 101 단계에서는 거창한 개념보다 "어느 폴더에서 시작하는지", "어떻게 작은 브랜치를 따는지", "결과를 어디서 보는지"가 더 중요하다.

## 필수

| 개념 | 한 줄 설명 | 추천 자료 |
|------|-----------|---------|
| 작업 폴더 | Codex는 현재 열어 둔 폴더 문맥을 기준으로 일한다. | [quick-start](docs/guides/quick-start.md) |
| 작은 브랜치 | main 대신 작은 feature branch에서 작업해야 비교와 롤백이 쉽다. | [faq](faq.md) |
| Diff 확인 | 결과를 바로 믿지 말고 변경 내용을 먼저 보는 습관이다. | [review-vs-implementation](docs/concepts/review-vs-implementation.md) |

## 권장

| 개념 | 한 줄 설명 | 추천 자료 |
|------|-----------|---------|
| Context 전달 | 목표, 범위, 금지선을 짧게 넘기면 결과가 훨씬 안정된다. | [agents-and-context](docs/concepts/agents-and-context.md) |
| 테스트 확인 | 변경 후 최소한의 검증을 같이 돌리면 불안이 줄어든다. | [quick-start](docs/guides/quick-start.md) |
