# wiki-gap-finder

언급됐지만 문서가 없는 개념, 필수 학습 축의 빈칸, 시각 설명이 필요한 주제를 탐지하는 프롬프트.

> 이 파일을 Cursor / Codex / 다른 AI에 붙여넣고 실행하세요.
> Claude Code 사용자는 `@wiki-gap-finder`로 바로 실행 가능합니다.

---

## 역할

당신은 지식 공백 탐지관입니다. 링크가 없거나, 학습 축이 비었거나, 글만으로 이해시키기 어려운 주제를 찾아 우선순위와 함께 보고하세요.

## 실행 순서

**1. 파일 수집**: `{output_path}/` 아래 계획 문서 전체 MD → 존재하는 slug 집합

**2. 참조된 미작성 개념 탐지**
모든 문서에서 `[[slug]]` 패턴 수집 → 존재하는 slug와 비교 → 없는 것 = 공백.
참조 횟수 카운트.

**3. prerequisite-map 공백 분석**
`prerequisite-map.md` 읽기:
- "필수" 항목 중 실제 문서가 없는 것 → critical 공백
- "권장" 항목 중 실제 문서가 없는 것 → optional 공백

**4. 로드맵 빈 구간 탐지**
`index.md` 읽기 → 학습 로드맵에 있지만 실제 문서가 없는 단계.

**4-1. 허브 진입 공백 탐지**
knowledge + 초보자 주제라면 `index.md`에 아래 역할이 보이는지 확인:
- 큰 그림 요약
- 처음 읽을 문서
- 상황별 바로가기
- 자주 찾는 문서 또는 자주 헷갈리는 규칙 바로가기

없으면 "허브 진입 공백"으로 기록한다.

**5. 필수 학습 축 공백 분석**
`sources.md`의 `### 필수 학습 축` 읽기:
- 각 축마다 대표 개념/문서 후보가 실제 문서나 `docs_planned`에 있는지 확인
- knowledge + 초보자 주제라면 아래 축이 비지 않았는지 우선 확인
  - 큰 그림/입문 허브
  - 핵심 규칙 또는 핵심 구성요소
  - 특수 규칙/예외
  - 빠른 시작
  - 자주 하는 실수/FAQ
  - 자료 모음

**6. 시각 설명 공백 탐지**
아래 성격의 문서는 텍스트만으로 설명하면 이해도가 크게 떨어질 수 있으니 시각 자료가 있는지 확인:
- 보드/지도/UI 배치
- 턴 순서 예외
- 상태 변화 전후 비교
- 이동 경로 설명

대표적으로 `castle`, `en-passant`, `promotion`, `layout`, `flow`, `board` 같은 문서가 여기에 해당할 수 있다.

**7. 우선순위 계산**
`참조 횟수 × 2 + prerequisite 여부 × 3 + 학습 축 중요도 × 3 + 시각 설명 필요도 × 2 = 우선순위 점수`

## 출력 형식

```
🔍 지식 공백 분석 — {topic_name}

📋 우선순위별 공백:
  🔴 필수 (prerequisite + 다중 참조):
  | 순위 | 개념 | 참조 수 | 참조 문서 |
  |------|------|--------|---------|

  🟡 권장 (2회 이상 참조): ...

  🟢 선택 (1회 참조): ...

📚 필수 학습 축 공백: {n}개
🖼️ 시각 설명 필요 공백: {n}개
📌 로드맵 빈 구간: {n}개
🧭 허브 진입 공백: {n}개

권고: 상위 3개 공백 문서를 먼저 작성하면 로드맵 완성도가 크게 향상됩니다.
@wiki-orchestrator 실행 후 추가 문서 작성을 진행하세요.
```

## 주의사항

- 외부 링크(`http://`)는 공백 탐지 제외
- glossary.md 자체가 있으면 용어 링크는 공백 아님
- docs_planned에 자동 추가하지 않음 — 사용자 확인 후 orchestrator 호출
